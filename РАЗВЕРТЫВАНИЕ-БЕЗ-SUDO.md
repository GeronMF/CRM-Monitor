# Развертывание CRM Monitor без sudo прав

## Вариант 1: Работа через администратора сервера

### Шаг 1: Подготовка проекта локально

1. **Соберите проект:**
   ```bash
   npm install
   # Создайте .env файл с вашими Supabase данными
   npm run build
   ```

2. **Создайте архив:**
   ```bash
   # На Windows
   # Заархивируйте папку dist/ и отправьте на сервер
   
   # Или на сервере (если есть доступ по SSH без sudo)
   tar -czf crm-monitor-dist.tar.gz dist/
   ```

### Шаг 2: Передача файлов администратору

Отправьте администратору сервера:

1. **Собранные файлы** из папки `dist/` (или архив)
2. **Фрагмент конфигурации Nginx** из файла `nginx-location-block.conf`
3. **Инструкцию для администратора** (см. ниже)

### Шаг 3: Инструкция для администратора сервера

**Текст для отправки администратору:**

```
Здравствуйте!

Нужно развернуть проект CRM Monitor в поддиректории /crm-monitor на домене ai.ecom-upload.space.

1. Скопируйте файлы из dist/ в /var/www/ai.ecom-upload.space/crm-monitor/
   (или в другую директорию, если у вас другая структура)

2. Добавьте в конфигурацию Nginx для ai.ecom-upload.space следующий блок:

[вставить содержимое nginx-location-block.conf]

3. Проверьте конфигурацию: nginx -t
4. Перезагрузите Nginx: systemctl reload nginx

Путь к файлам на сервере: /var/www/ai.ecom-upload.space/crm-monitor/
```

## Вариант 2: Работа в домашней директории (если доступна)

Если у вас есть доступ к веб-серверу через домашнюю директорию (например, `~/public_html` или `~/www`):

### Шаг 1: Определите путь к веб-директории

```bash
# Обычно это одна из этих директорий:
# ~/public_html
# ~/www
# ~/html
# ~/domains/ai.ecom-upload.space/public_html

# Проверьте, какая директория доступна для веб-сервера
ls -la ~/
```

### Шаг 2: Создайте структуру директорий

```bash
# Если у вас есть доступ к корневой веб-директории домена
mkdir -p ~/public_html/crm-monitor

# Или если структура другая, уточните у администратора
```

### Шаг 3: Соберите и скопируйте проект

```bash
# На вашем компьютере
npm install
# Создайте .env с Supabase данными
npm run build

# Скопируйте файлы на сервер (через SCP, FTP, или другой способ)
# scp -r dist/* user@server:~/public_html/crm-monitor/
```

### Шаг 4: Попросите администратора добавить конфигурацию

Отправьте администратору фрагмент из `nginx-location-block.conf`, но с правильным путем:

```nginx
location /crm-monitor {
    alias /home/ваш-username/public_html/crm-monitor;  # Укажите правильный путь
    try_files $uri $uri/ /crm-monitor/index.html;
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    location ~* \.html$ {
        expires -1;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }
}
```

## Вариант 3: Через панель управления хостингом

Если у вас есть панель управления (cPanel, Plesk, ISPmanager и т.д.):

### Шаг 1: Загрузите файлы

1. Войдите в панель управления
2. Откройте файловый менеджер
3. Перейдите в директорию домена (обычно `public_html` или `www`)
4. Создайте папку `crm-monitor`
5. Загрузите все файлы из `dist/` в эту папку

### Шаг 2: Настройка через панель

**Для cPanel:**
- Некоторые панели позволяют настроить поддиректории автоматически
- Или используйте "Дополнительные домены" / "Поддомены"

**Для других панелей:**
- Обратитесь в поддержку хостинга с запросом настроить `/crm-monitor` для вашего домена

## Проверка после развертывания

После того как администратор настроит Nginx:

1. Откройте: `https://ai.ecom-upload.space/crm-monitor`
2. Проверьте консоль браузера (F12) на ошибки
3. Убедитесь, что статические файлы загружаются (проверьте Network вкладку)

---

## Обновление проекта (без sudo)

Когда нужно обновить проект:

1. **Локально:**
   ```bash
   npm run build
   ```

2. **Загрузите новые файлы:**
   - Через FTP/SFTP замените файлы в `crm-monitor/`
   - Или отправьте архив администратору

3. **Очистите кэш браузера** (Ctrl+Shift+R)

## Важные замечания

- ✅ Проект уже настроен для работы в `/crm-monitor`
- ✅ Все пути в коде корректны
- ✅ Нужна только конфигурация Nginx от администратора
- ✅ Файлы можно разместить в любой доступной вам директории

## Если что-то не работает

1. Проверьте, что файлы загружены в правильную директорию
2. Убедитесь, что администратор добавил конфигурацию Nginx
3. Проверьте права доступа к файлам (должны быть читаемыми)
4. Проверьте логи Nginx (попросите администратора)
