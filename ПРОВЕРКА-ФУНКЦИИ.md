# Проверка функции parse-crm-data

## Проблема
Данные не меняются при смене сессии, хотя запросы идут с правильными параметрами.

## Что проверить:

### 1. Функция обновлена в Supabase?
- Откройте: https://supabase.com/dashboard/project/awangomqhrqmljoetgta/functions
- Найдите функцию `parse-crm-data`
- Проверьте, что код содержит:
  - `const sessionParam = url.searchParams.get('session')`
  - `const session = sessionParam.toLowerCase()`
  - `const config = sessionConfigs[session]`
  - Логирование с `console.log('Session:', session)`

### 2. Проверьте логи в Supabase
- Dashboard → Edge Functions → parse-crm-data → Logs
- Должны быть сообщения:
  - `=== FUNCTION CALLED ===`
  - `Session parameter received: diar` (или rozpakuj)
  - `Using config for session: diar`
  - `Login: Zaharchenko` (для diar/rozpakuj)

### 3. Если логи пустые
Возможные причины:
- Функция не обновлена (старая версия)
- Логи не сохраняются
- Функция не вызывается

### 4. Проверка через debug режим
В консоли браузера добавьте `?debug=true`:
```
https://awangomqhrqmljoetgta.supabase.co/functions/v1/parse-crm-data?session=diar&debug=true
```

### 5. Ручная проверка функции
Скопируйте ВЕСЬ код из `supabase/functions/parse-crm-data/index.ts` и:
1. Откройте Supabase Dashboard
2. Edge Functions → parse-crm-data → Edit
3. Удалите весь старый код
4. Вставьте новый код
5. Нажмите Deploy
6. Дождитесь завершения деплоя

### 6. Проверка после обновления
1. Откройте приложение
2. Выберите сессию "ДИАР"
3. Откройте консоль браузера (F12)
4. Проверьте логи Supabase
5. Должны быть разные данные
