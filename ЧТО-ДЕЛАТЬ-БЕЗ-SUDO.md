# Что делать, если нет sudo прав

## Ваш план действий:

### 1. Подготовьте проект локально

```bash
# Установите зависимости
npm install

# Создайте файл .env (скопируйте из env.example.txt)
# Заполните ваши Supabase данные:
# VITE_SUPABASE_URL=https://ваш-проект.supabase.co
# VITE_SUPABASE_ANON_KEY=ваш-ключ

# Соберите проект
npm run build
```

### 2. Подготовьте файлы для администратора

После сборки у вас будет папка `dist/` с готовыми файлами.

**Что нужно передать администратору:**

1. ✅ **Все файлы из папки `dist/`** (можно заархивировать)
2. ✅ **Файл `ИНСТРУКЦИЯ-ДЛЯ-АДМИНИСТРАТОРА.txt`** - готовый текст для администратора
3. ✅ **Файл `nginx-location-block.conf`** - фрагмент конфигурации

### 3. Отправьте администратору

**Вариант А: Через файловый менеджер/FTP**
- Загрузите файлы из `dist/` в доступную вам директорию
- Отправьте администратору инструкцию и попросите настроить Nginx

**Вариант Б: Через администратора**
- Отправьте архив с файлами из `dist/`
- Отправьте файл `ИНСТРУКЦИЯ-ДЛЯ-АДМИНИСТРАТОРА.txt`
- Администратор сам разместит файлы и настроит Nginx

### 4. Что должен сделать администратор

Администратор должен:

1. Скопировать файлы из `dist/` в `/var/www/ai.ecom-upload.space/crm-monitor/`
2. Добавить конфигурацию Nginx (фрагмент из `nginx-location-block.conf`)
3. Перезагрузить Nginx

**Текст для отправки администратору:**

```
Здравствуйте!

Нужно развернуть проект CRM Monitor в поддиректории /crm-monitor 
на домене ai.ecom-upload.space

Прикрепляю:
1. Файлы проекта (папка dist/)
2. Инструкцию (ИНСТРУКЦИЯ-ДЛЯ-АДМИНИСТРАТОРА.txt)
3. Фрагмент конфигурации Nginx (nginx-location-block.conf)

Проект должен быть доступен по адресу: https://ai.ecom-upload.space/crm-monitor

Спасибо!
```

## Альтернативный вариант: через панель управления

Если у вас есть доступ к панели управления хостингом (cPanel, Plesk и т.д.):

1. Войдите в панель управления
2. Откройте файловый менеджер
3. Найдите директорию домена (обычно `public_html` или `www`)
4. Создайте папку `crm-monitor`
5. Загрузите все файлы из `dist/` в эту папку
6. Обратитесь в поддержку с просьбой настроить маршрутизацию для `/crm-monitor`

## После развертывания

Проверьте:
- ✅ Откройте `https://ai.ecom-upload.space/crm-monitor`
- ✅ Проверьте консоль браузера (F12) на ошибки
- ✅ Убедитесь, что статические файлы загружаются

## Обновление проекта

Когда нужно обновить:

1. Локально: `npm run build`
2. Замените файлы в `dist/` на сервере (через FTP/файловый менеджер)
3. Очистите кэш браузера (Ctrl+Shift+R)
