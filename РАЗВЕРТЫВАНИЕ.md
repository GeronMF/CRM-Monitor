# Быстрая инструкция по развертыванию CRM Monitor

> **⚠️ Если у вас нет sudo прав**, см. файл `РАЗВЕРТЫВАНИЕ-БЕЗ-SUDO.md`

## Подготовка

1. **Скопируйте проект на сервер:**
   ```bash
   # На сервере
   sudo mkdir -p /var/www/ai.ecom-upload.space/crm-monitor
   # Скопируйте все файлы проекта в эту директорию
   ```

2. **Установите зависимости:**
   ```bash
   cd /var/www/ai.ecom-upload.space/crm-monitor
   npm install
   ```

3. **Создайте файл .env:**
   ```bash
   cp env.example.txt .env
   nano .env
   ```
   Заполните:
   - `VITE_SUPABASE_URL` - URL вашего Supabase проекта
   - `VITE_SUPABASE_ANON_KEY` - публичный ключ Supabase

## Настройка базы данных

### Вариант 1: Supabase (рекомендуется)

1. Войдите в Supabase проект
2. SQL Editor → выполните миграцию из `supabase/migrations/20251209185430_create_crm_tables.sql`
3. Разверните Edge Function:
   ```bash
   npm install -g supabase
   supabase login
   supabase link --project-ref ваш-project-ref
   supabase functions deploy parse-crm-data
   ```

### Вариант 2: Локальная MySQL через phpMyAdmin

1. Откройте `https://ai.ecom-upload.space/phpmyadmin`
2. Создайте БД
3. Импортируйте `supabase/migrations/20251209185430_create_crm_tables.sql`

## Настройка Nginx

### Если у вас уже есть конфиг для ai.ecom-upload.space:

Добавьте в блок `server`:

```nginx
location /crm-monitor {
    alias /var/www/ai.ecom-upload.space/crm-monitor/dist;
    try_files $uri $uri/ /crm-monitor/index.html;
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    location ~* \.html$ {
        expires -1;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }
}
```

### Или используйте готовый конфиг:

```bash
sudo cp nginx.conf /etc/nginx/sites-available/ai.ecom-upload.space
# Отредактируйте пути и SSL сертификаты при необходимости
sudo nginx -t
sudo systemctl reload nginx
```
